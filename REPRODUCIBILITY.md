# Reproducibility

Every number/figure is regenerated by the commands below. No data is vendored; device costs are measured.

## Environment
- Python ≥ 3.9; `pip install -e .` (numpy; dev extras: pytest, matplotlib, scipy).
- Synthetic results are deterministic (fixed seeds). Real-arm results depend on the measured device.
- Published real-arm hardware: AWS EC2 **c5d.2xlarge** spot (8 vCPU), Ubuntu 22.04, ap-south-1, with an
  **EBS gp3** root volume and a mounted **instance-store NVMe**; **PostgreSQL 14**; torn down after the run.

## Synthetic (laptop, < 30 s) — H0, H1, threshold, open-loop contrast + NCs
```
pip install -e .
pytest -q                       # 14 tests (closed forms, NCs, gate)
python bench/run_synthetic.py   # results/{H1,threshold,contrast}.json, figures/{fixedpoint,threshold,contrast}.png
```

## Real device fsync model (run on the target filesystem)
```
python bench/measure_fsync.py --path /home/ubuntu/fsynctest.dat --reps 3000 --device-label ebs-gp3 --out results/fsync_ebs.json
python bench/measure_fsync.py --path /mnt/nvme/fsynctest.dat     --reps 3000 --device-label instance-nvme --out results/fsync_nvme.json
```

## Real PostgreSQL commit_delay sweep
```
sudo apt-get install -y postgresql
bench/pg_commit_delay.sh results/pg_commit_delay.json 32 20 50   # clients, seconds, scale
bench/pg_commit_delay.sh results/pg_c2.json            2  20 50   # low-load (below lambda*)
```

## Combine (real arm)
```
python bench/run_real.py        # results/H5.json, figures/devices.png
```

## Provenance
- fsync JSON records device label, p50/p90/p99/mean per write size, F0 sample distribution, and δ.
- PostgreSQL: fsync=on, synchronous_commit=on; pgbench TPC-B-like; tps/latency per commit_delay.
- Seeds fixed in `group_commit/experiment.py`; offline DP verified vs brute force in tests.
